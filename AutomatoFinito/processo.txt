Este projeto representa uma versão simplificada do que seria um controlador de requisições automatizadas, 
normalmente utilizado dentro de servidores de webhooks, serviços de scraping ou serviços de despacho de atividades e marketing em centrais.

O sistema recebe um conjunto de instruções que pode ou não formar um loop e uma instrução de inicialização. Ao inicializar 
o software requisitará um ponto da internet e salvará as informações em algum local predefinido.

 -> ALFABETO DE ENTRADA:

    Método de requisição: GET; POST; PUT; DELETE;
    URL: destino da requisição - ex: "https://api.joinrise.io/api/v1/jobs/public?page=1&limit=20&sort=desc&sortedBy=createdAt&jobLoc=";
    Headers: se houver, definir lista de conjunto de key e value;
    Body: objeto do body da requisição em formato stringfyied;

 -> CONFIGURAÇÕES DE ENTRADA: 

    Repetir na falha: Não?; Sim? -> Quantos?;
    Gravar respostas: Sim? Não?;
    Timeout: Não? Sim? -> Segundos?;
    Delay entre requisições? Não? Sim? -> Segundos?;

 -> ESTADOS: 

    Abertura;
    (CICLO BASE)
    (CICLO CONFIGURAÇÕES)

    (CLICO BASE):
        Definição de método -> Definição da URL -> Tem header? (CICLO CADASTRO HEADER) -> Definição do body -> [Confirma Informações] -> Sim -> OK;
                                                                                                                                      -> Não -> Menu numérico de passo -> Abre definição -> [Confirma Informações];

    (CICLO CADASTRO HEADER):
        Defina key -> Defina value -> Mostra item -> Adicionar mais; Concluir Headers; Editar último item; -> OK
